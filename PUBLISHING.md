# Publishing ScrollBrake

This guide covers the exact steps to prepare and submit ScrollBrake to the Chrome Web Store and Firefox Add-ons.

## Before You Submit

- Pick a stable version number and update it in `package.json`
- For Firefox, replace the temporary Gecko ID in `scripts/build.mjs` with a stable public ID you control, such as `scrollbrake@yourdomain.com`
- Make sure the extension name, description, and privacy claims are consistent across the repo, store listings, and privacy policy
- Make sure the system `zip` command is available locally because the packaging step uses it to create store upload archives

## Build And Package

Run these commands from the project root:

```bash
npm run check
npm run build
npm run package
```

Artifacts created by the packaging step:

- `release/scrollbrake-chrome-<version>.zip`
- `release/scrollbrake-firefox-<version>.zip`

The ZIP files are created from the existing `dist/` build outputs with the extension files at the archive root, which is what the stores expect.

## Chrome Web Store Checklist

1. Create a Chrome Web Store developer account and pay the one-time registration fee.
2. Open the developer dashboard and create a new item.
3. Upload `release/scrollbrake-chrome-<version>.zip`.
4. Complete the Store Listing tab.
5. Complete the Privacy tab.
6. Set visibility and distribution.
7. Add test instructions for the reviewer.
8. Submit for review.

### Chrome Listing Draft

Short description:

> Limit time on distracting sites with local-only usage tracking and timed lockouts.

Long description:

> ScrollBrake helps you spend less time on distracting websites by giving each site a time budget. When the budget runs out, the site is locked for a short break.
>
> Features:
>
> - Built-in controls for Facebook, Instagram, X, and YouTube
> - Support for custom domains
> - Configurable usage limit and break duration
> - Tracks active usage only while the page is visible and focused
> - Popup dashboard plus full settings page
> - Local-only storage with no analytics, accounts, or sync
>
> ScrollBrake is designed to be simple, private, and easy to adjust throughout the day.

Single purpose:

> Help users limit time spent on selected websites by tracking active usage locally and temporarily blocking those sites after a configured limit.

Permissions justification:

- `storage`: saves the user’s settings and timer state on the local device
- `tabs`: reads the active tab URL so the popup can show the current site status
- host permissions on all URLs: allows users to track built-in sites and any custom domains they choose to add

Privacy/data use:

- No remote code
- No analytics
- No account system
- No browsing history sent to a server
- Settings and timer state remain on the local device
- Privacy policy URL: publish `PRIVACY.md` at a public URL and use that link in the dashboard

Reviewer notes:

> The extension is fully local-only. It uses `storage` for settings/timer state, `tabs` to inspect the active tab for the popup, and broad host access so users can add arbitrary custom domains to monitor. No code or data is loaded from remote servers.

### Chrome Assets To Prepare

- 128x128 icon
- At least one 1280x800 screenshot
- 440x280 small promo tile
- Optional 1400x560 marquee promo tile

Suggested screenshots:

- Popup showing enabled sites and remaining time
- Options page showing custom site setup
- Full-page lock overlay after the timer runs out

## Firefox Add-ons Checklist

1. Create an AMO developer account.
2. Create a new add-on listing in the Developer Hub.
3. Upload `release/scrollbrake-firefox-<version>.zip`.
4. Choose listed distribution on AMO.
5. Upload source code when prompted so reviewers can reproduce the build.
6. Complete the listing, categories, and reviewer notes.
7. Submit for review.

### Firefox Listing Draft

Summary:

> Limit time on distracting sites with local-only timers and automatic break locks.

Description:

> ScrollBrake helps you set time budgets for distracting sites and automatically locks them for a short break when the limit is reached.
>
> It includes built-in support for Facebook, Instagram, X, and YouTube, and it also lets you add your own custom domains.
>
> ScrollBrake tracks active usage only while a page is visible and focused. All settings and timer data stay in Firefox’s local extension storage on the current device.
>
> Highlights:
>
> - Per-site usage limits
> - Configurable lockout duration
> - Support for custom domains
> - Popup dashboard and full settings page
> - Full-page countdown overlay during lockout
> - No analytics, accounts, or server sync

Reviewer notes:

> Build steps:
>
> 1. Run `npm run build:firefox`
> 2. Run `npm run package:firefox`
>
> The Firefox manifest is generated by `scripts/build.mjs`. The add-on is local-only and does not make network requests. Broad host access is required because users can add any domain they want to track and lock.

Source code submission notes:

- Upload a source archive from the repository root
- Include the original source files plus the build and packaging scripts in `scripts/`
- Exclude generated `dist/` and `release/` artifacts if AMO asks for source rather than built output

## Privacy Policy

Use the text in `PRIVACY.md` as the basis for your store privacy disclosures. If you host the repository on GitHub, you can point the store listing to the public file URL or publish the same text on your website.

## Release Flow

1. Update the version.
2. Replace the Firefox Gecko ID if needed.
3. Run `npm run check`.
4. Run `npm run build`.
5. Run `npm run package`.
6. Test the packaged build in Chrome and Firefox one last time.
7. Upload the generated ZIP to each store.
